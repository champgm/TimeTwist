# Unit Test Coverage Plan Pack

## Scope Summary
- Expand the current near-empty `app/src/test/java/` suite into a reliable regression harness for timer behavior.
- Keep the first implementation pass inside JVM unit tests run by `./gradlew testDebugUnitTest`.
- Add Android-aware unit testing with `Robolectric` so `Context`, `SharedPreferences`, `Application`, and `Service` behavior can be exercised without `androidTest`.
- Add `kotlinx-coroutines-test` so timer loops and `viewModelScope` work can be advanced deterministically.
- Preserve the existing single-module architecture under `presentation/`, `service/`, and `ui/`.
- Prefer small testability seams over broad refactors: extract time sources or helper logic only where the current code blocks deterministic tests.
- Cover the highest-risk behavior first: preference persistence, `TimerViewModel` state transitions, and `CountdownService` alert cadence.
- Expand existing slider math tests rather than replacing them.
- Defer emulator-backed Compose UI tests until after the JVM suite is stable.
- Add lightweight coverage reporting only after the new tests exist and are green.

## Phases
- `01_test_harness_and_seams.md`
  Establish the test stack, add deterministic test seams, and keep the app behavior unchanged while making Android-backed unit tests practical.
- `02_pure_logic_and_prefs.md`
  Build fast regression tests for slider math, time-formatting extensions, and `SharedPreferences` persistence helpers.
- `03_timer_viewmodel.md`
  Cover `TimerViewModel` loading, updates, service-launch intent contents, stopping behavior, and completion/repeat logic with controlled time and coroutine scheduling.
- `04_countdown_service_and_coverage.md`
  Cover `CountdownService` countdown rules and alert cadence, then add coverage reporting and a short research note documenting the new test strategy.

## Cross-Phase Conventions
- Naming
  - Keep production code names aligned with existing packages under `com.cgm.timetwist`.
  - Name new unit test files `*Test.kt` and colocate them under `app/src/test/java/com/cgm/timetwist/...`.
  - If helper fakes are needed, prefer `Fake*` or `Test*` names inside the relevant test package.
- Config patterns
  - Update only [app/build.gradle.kts](/mnt/nvme/github/TimeTwist/app/build.gradle.kts) for test-library additions and coverage plugin configuration.
  - Do not introduce new modules, product flavors, or `androidTest` source sets in this plan.
  - If a new shared test utility is needed, keep it under `app/src/test/java/com/cgm/timetwist/` rather than adding a separate Gradle source set.
- Logging and test conventions
  - Prefer assertions on observable state, intents, notifications, and persisted values rather than log strings.
  - Use `Robolectric` shadows where Android framework behavior must be inspected.
  - Use `kotlinx-coroutines-test` schedulers or explicit time providers instead of `Thread.sleep`.
  - Keep all validation runnable from repo root with `./gradlew testDebugUnitTest`.

## Global Definition Of Done
- [ ] `./gradlew testDebugUnitTest` passes from repo root with the new suite.
- [ ] The suite covers pure logic, persistence helpers, `TimerViewModel`, and `CountdownService`.
- [ ] Added seams do not change app behavior or public navigation structure.
- [ ] No emulator or device is required for the first-pass regression suite.
- [ ] Coverage reporting is generated by a documented Gradle task and stays aligned with the new tests.
- [ ] Any new helper abstractions are minimal, named clearly, and used only to enable deterministic tests.
- [ ] Documentation or research notes explain the chosen JVM-plus-Robolectric strategy for future contributors.
